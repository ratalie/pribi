NOMBRE DEL PROYECTO
-------------------
v0-double-sidebar-component

DESCRIPCIÓN
-----------
Aplicación web de documentación con doble sidebar (navegación lateral izquierda 
y tabla de contenidos derecha) construida con Next.js 16 y React 19. 

El proyecto simula un sistema de gestión de juntas generales empresariales con 
múltiples secciones y flujos de trabajo documentados.

PROPÓSITO PRINCIPAL
-------------------
Proporcionar una interfaz de documentación navegable con:
- Navegación jerárquica por secciones (sidebar izquierdo)
- Tabla de contenidos contextual (sidebar derecho)
- Estructura de wizard/formulario multi-paso
- Visualización de información sobre juntas empresariales

CARACTERÍSTICAS CLAVE
---------------------
✓ Doble sidebar responsivo
✓ Navegación contextual automática
✓ Scroll suave a secciones
✓ Route groups de Next.js
✓ Tema claro/oscuro
✓ Mobile-first responsive design
✓ TypeScript strict mode
✓ Tailwind CSS v4

STACK TECNOLÓGICO PRINCIPAL
----------------------------
Framework:          Next.js 16.0.0 (App Router)
Librería UI:        React 19.2.0
Lenguaje:           TypeScript 5
Estilos:            Tailwind CSS 4.1.9
Componentes:        shadcn/ui (sistema de componentes)
Primitivos UI:      Radix UI (componentes accesibles)
Gestión de temas:   next-themes
Analytics:          Vercel Analytics
Tipografías:        Geist & Geist Mono (Google Fonts)

ARQUITECTURA
------------
Tipo: Single Page Application (SPA) con Server Components
Patrón: App Router de Next.js con Route Groups
Routing: File-based routing jerárquico
State: Local state con React hooks (sin Redux/Zustand)
Styling: Utility-first con Tailwind CSS

ESTRUCTURA DE NAVEGACIÓN
-------------------------
1. Selección de Junta
2. Datos de la Junta
3. Instalación de la Junta
4. Puntos de Agenda (6 sub-opciones)
   - Aumento de Capital por Aporte
   - Aumento de Capital por Capitalización
   - Nombramiento de Gerente
   - Nombramiento de Apoderados
   - Remoción de Apoderados
   - Remoción de Directores
5. Resumen
6. Descargar Documentos

COMPONENTES CLAVE
------------------
- DocsLayout: Layout principal con doble sidebar
- MainSidebar: Navegación principal izquierda
- TableOfContents: Navegación contextual derecha
- Button: Único componente UI de shadcn/ui en uso

CARACTERÍSTICAS TÉCNICAS
-------------------------
- Server Components por defecto
- Client Components donde necesario ('use client')
- Path aliases (@/ para imports)
- CSS Variables para theming
- Mobile overlay navigation
- Scroll anchor links
- Nested layouts

DEPENDENCIAS NOTABLES
---------------------
